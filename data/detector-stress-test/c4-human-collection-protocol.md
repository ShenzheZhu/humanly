# C4 Human-Written AI-Style Collection Protocol

Purpose: collect the `C4` case for the detector stress test.

`C4` is the compliant false-positive-risk case: a human writes in a formal,
AI-associated style without using AI generation. The final text may look
detector-suspicious, but the process label is still compliant because the text
is human-written.

Current collection route: recruit writers through Prolific and collect direct
text-entry responses in Prolific or a linked survey. These samples are
self-attested human-origin final texts for detector false-positive testing; they
should not be described as Humanly process-traced samples.

## Required Sample Count

- 10 short samples: social media post style.
- 10 medium samples: student assignment response style.
- 10 long samples: paper review style.
- 30 total samples.

Each sample maps to an existing target id:

- `c4_short_01` through `c4_short_10`
- `c4_medium_01` through `c4_medium_10`
- `c4_long_01` through `c4_long_10`

The executable collection manifest is generated by:

```bash
node data/detector-stress-test/scripts/build_c4_collection_manifest.mjs
```

It writes `c4-human-collection-manifest.csv`, which maps every target id to its
length bucket, task type, source prompt path, target writer-input path, and
final import path.

The Prolific study packet is generated by:

```bash
node data/detector-stress-test/scripts/build_prolific_c4_study_pack.mjs
```

It writes `prolific/c4-writing-items.csv`,
`prolific/c4-writing-budget-estimate.csv`, and draft worker instructions.

## Writer Instructions

Writers should:

1. Write the sample themselves without AI generation.
2. Use an intentionally formal, AI-associated style.
3. Include some of the AI-associated vocabulary from the prompt packet when it
   fits naturally.
4. Stay within the assigned length bucket:
   - short: roughly 120-180 words
   - medium: roughly 400-600 words
   - long: roughly 1000-1500 words
5. Avoid copying text from published sources.
6. Avoid pasting AI-generated text and editing it.

Acceptable assistance:

- Spell-checking without text generation.
- Looking at the C4 instruction and vocabulary list.
- Reading the matched task prompt.

Not acceptable:

- Asking an LLM to draft, rewrite, humanize, translate, or polish the sample.
- Using a paraphraser or humanizer.
- Copying a published passage and making surface edits.

## Storage Format

Save collected files under:

```text
data/detector-stress-test/texts/human_c4/
```

with filenames:

```text
c4_short_01.txt
c4_short_02.txt
...
c4_long_10.txt
```

Then run:

```bash
node data/detector-stress-test/scripts/import_c4_human_samples.mjs
node data/detector-stress-test/scripts/build_case_generation_jobs.mjs
node data/detector-stress-test/scripts/export_generated_samples.mjs
node data/detector-stress-test/scripts/audit_paper_ready_gates.mjs
node data/detector-stress-test/scripts/validate_dataset.mjs
```

## Metadata

For each writer/session, keep a private note outside the public dataset with:

- writer id or pseudonym,
- date collected,
- whether Humanly logs exist,
- whether the text may be stored in the repo,
- whether the text may be released publicly,
- any compensation/consent note.

Do not put personal information in the public manifest.
